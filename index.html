Nightfever

Nightfever is an international non-profit organisation consisting of a large amount of local groups that organize events. They open churches during the night and invite passersby to a moment of tranquility and encounter.

At the moment Nightfever is regularly held in about 100 cities world wide.

Motivation

  • Our website is digitally outdated
  • Big challenges: Internationalization, mobile use
  • Increase features and usability
  • Software is hard to maintain and extend
  • Security and reliability threatened

Our web strategy needs a new approach!

Goals

  • Reliable and future proof platform
  • Modern design, mobile-ready
  • Internationalization support
  • High Usability for visitors and editors

User Stories

User Stories

A: A user who looks for general information on nightfever wants to …

  1. get a precise description of the idea and purpose of Nightfever
  2. be able to learn more about it
  3. discover nightfever cities and countries
  4. learn about nearby locations and dates

User Stories (Cont.)

B: A user who looks for information on a specific nightfever city wants to …

  1. find upcoming events in that city
  2. consume articles and media about previous events in that city
  3. learn more about nightfever
  4. discover nearby cities

User Stories (Cont.)

C: A regular user looking for updates on Nightfever wants to …

  1. get the latest news (articles, media, newsletters)
  2. know about upcoming important events

D: A regular user looking for updates on a specific city wants to …

  1. find upcoming events in that city and nearby cities
  2. read latest news from that city
  3. learn about important news from other cities and global

User Stories (Cont.)

E: A regular user looking for updates on multiple city wants to …

  1. same as D
  2. navigate between cities

F: A user who wants to participate or start Nightfever in his city wants to …

  1. get background information on nightfever
  2. know how they can participate

Website Structure

Scopes

  • a scope acts as an independent website on its own domain
  • creates an individual hierarchy
  • contains content elements and aggregates them
  • user permissions are given on a scope domain

Landing Pages

Each scope has a landing page, plus there are a few extra landing pages inside the global scope.
  • global site nightfever.org
  • city sites (local nightfever groups) [city].nightfever.org
  • project sites (teams & major events) [project].nightfever.org
  • regional sites (countries) nightfever.org/[region]
  • micro-sites (nightfever specials) nightfever.org/special

Landing Pages (Cont.)

Landing pages are customizabe using common elements:

  • news feed
  • navigation
  • teasers/news
  • upcoming events
  • media elements
  • announcements

Content Types

  • Page static webpage
  • Story full-featured articles with enhanced authoring features
  • Event several event types
  • Micropost short text post with the ability to reference internal or external content including a preview of the referenced content
  • Venue event locations of Nightfever evenings
  • Media & Gallery foto, video and audio elements
  • Statement statements from visitors

Web Navigation

Navigation Features

  1. subpages and other content in the same scope
  2. quick access to all other cities and projects
  3. switching between different localizations of the current page (if available)
  4. related entities of a certain site (country -> cities, event -> city)
  5. discover nearby nightfever cities
  6. upcoming events (including but not limited to nearby cities)

Navigation Layers

Two seperate layers of navigation:

  • Global navigation allows to directly navigate to different sites
  • Local navigation to reach pages inside a single site.

Both navigation concepts should be available from any page.

Global Navigation

Direct access to all other scopes from any page. To cover the amount of navigation targets (~100 cities) implemented as autosuggest text field.

The suggestions can easily enriched with metadata such as the distance from the users location or the next date.

This text field may also serve the purpose of a search field. Examples for this combination are at Facebook or Google Mail, where the search field suggest important navigation targets.

Global Navigation

Partial Localization Problem

  • Most scopes only provide one localization, maybe two or three
  • Example: Nightfever Bamberg will most likly not provide russian content
  • Importance primarily for global navigation, local navigation should be coherent
  • Possible solution: Provide auto-generated stub landing page for the users current language
